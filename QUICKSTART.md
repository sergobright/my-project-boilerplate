# Universal AI SaaS Boilerplate — Agent Quickstart Guide

**ВНИМАНИЕ АГЕНТУ:** Если ты читаешь этот файл, твоя задача — с нуля развернуть новый проект на базе "Universal AI SaaS Boilerplate". Действуй строго по шагам ниже. Не пропускай шаги и сообщай пользователю о каждом выполненном этапе.

## Шаг 1: Скачивание и инициализация репозитория
1. Тебе предоставлена эта сборка как стартовая точка. Если проект клонирован на новый сервер, переходи к следующему пункту.
2. Запусти скрипт `node scripts/init-project.js`. Запроси у пользователя имя проекта, чтобы скрипт выполнил автозамену старого имени (`HexaCash`) на новое во всех файлах (включая `package.json`, `docker-compose.yml`, `prisma/schema.prisma`).
3. Убедись, что скрипт очистил файлы `memory-bank/activeContext.md` и `memory-bank/progress.md`.
4. Убедись, что скрипт сгенерировал случайный `NEXTAUTH_SECRET` и создал `.env`.

## Шаг 2: Интервью с пользователем (Заполнение Memory Bank)
Текущие файлы `memory-bank/projectbrief.md` и `memory-bank/productContext.md` содержат старое описание (HexaCash) **исключительно для примера**.
Тебе нужно собрать полную информацию о новом проекте.
**ОСТАНОВИСЬ** и попроси пользователя:
*"Пожалуйста, предоставь максимально подробное и объемное описание нового проекта. Мне нужно понимать продукт максимально глубоко"*

Задай пользователю наводящие вопросы одним списком, если нужно:
1. Какова основная цель продукта и его ключевой функционал?
2. Кто целевая аудитория и какие их главные боли мы решаем?
3. Какая планируется воронка продаж (какие статусы)?
4. Какие внешние интеграции необходимы (Avito, Telegram, AmoCRM и т.д.)?

*Не продолжай развертывание, пока пользователь не предоставит подробное описание. После получения описания полностью перепиши `projectbrief.md`, `productContext.md` и `systemPatterns.md` под новый проект.*

## Шаг 3: Развёртывание инфраструктуры (Docker)
Когда Memory Bank полностью переписан под новый проект, подними инфраструктуру:
1. Выполни команду `docker compose up -d` для запуска контейнеров (БД, приложение).
2. Если ты работаешь на удаленном сервере, настрой проброс портов или nginx по указаниям пользователя.

## Шаг 4: Установка зависимостей и БД
Выполни команды в терминале:
1. `pnpm install` (установка зависимостей).
2. `pnpm --filter web db:generate` (генерация Prisma Client).
3. `pnpm --filter web db:migrate` (применение миграций БД).

## Шаг 5: Использование встроенных Скиллов (Skills)
* В папке `skills/` находятся bash/python скрипты (Dev Tools), которые ты (ИИ-агент) можешь вызывать для автоматизации своей работы в IDE. Обязательно изучи содержимое этой папки!
* В папке `apps/web/src/ai/skills/` находятся бизнес-скиллы для оркестратора (JSON-схемы для работы логики продукта). Не путай их со скиллами для IDE.

**ТВОЕ ПЕРВОЕ ДЕЙСТВИЕ СЕЙЧАС:**
Поздоровайся с пользователем, скажи, что ты прочитал `QUICKSTART.md`, готов инициализировать проект и запроси новое имя проекта для запуска `init-project.js`.
