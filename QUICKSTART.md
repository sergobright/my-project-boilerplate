# Agent Quickstart Guide

**ВНИМАНИЕ АГЕНТУ:** Если ты читаешь этот файл, твоя задача — с нуля развернуть новый B2B SaaS проект на базе готового стартового шаблона. Действуй строго по шагам ниже. Не пропускай шаги и сообщай пользователю о каждом выполненном этапе.

## Шаг 1: Скачивание репозитория
1. Выполни клонирование репозитория базового шаблона в текущую папку:
   `git clone https://github.com/sergobright/my-project-boilerplate .`
   *(Примечание: если папка не пуста, то удали все файлы в ней и замени новыми из репозитория).*

## Шаг 2: Инициализация нового проекта
1. Запусти скрипт `node scripts/init-project.js`. 
2. Запроси у пользователя новое имя проекта, чтобы скрипт выполнил автозамену старого имени (`HexaCash`) на новое во всех файлах (включая `package.json`, `docker-compose.yml`, `prisma/schema.prisma`).
3. Убедись, что скрипт сгенерировал новый `.env` файл с `NEXTAUTH_SECRET`.

## Шаг 3: Технологический стек и инфраструктура
Учти, что этот шаблон использует следующий стек:
- **Фронтенд и Бэкенд:** Next.js (App Router), TypeScript, TailwindCSS, Shadcn UI.
- **База данных:** PostgreSQL.
- **ORM:** Prisma (`prisma/schema.prisma`).
- **Авторизация:** NextAuth v5 (Auth.js).
- **Инфраструктура:** Docker Compose (контейнеры для БД и Next.js приложения).
- **Оркестрация AI:** N8N (внешние вебхуки приходят на `api/webhooks/n8n/route.ts`).

## Шаг 4: Интервью с пользователем (Заполнение Memory Bank)
Файлы в папке `memory-bank/` сейчас содержат старое описание проекта (HexaCash) исключительно для примера. Тебе нужно собрать информацию о новом проекте.
**ОСТАНОВИСЬ** и попроси пользователя:
*"Пожалуйста, предоставь максимально подробное описание нового проекта. Мне нужно понимать продукт так же глубоко, как это было сделано в базовом шаблоне."*

Задай пользователю наводящие вопросы:
1. Какова основная цель продукта и его ключевой функционал?
2. Кто целевая аудитория?
3. Какая планируется воронка продаж (какие статусы в БД)?
4. Какие внешние интеграции необходимы?

*После получения ответов полностью перепиши `memory-bank/projectbrief.md`, `memory-bank/productContext.md` и `memory-bank/systemPatterns.md` под новый проект.*

## Шаг 5: Запуск проекта
Поскольку проект работает полностью в Docker (база данных и само приложение), все команды нужно выполнять внутри контейнера.
Выполни следующие команды по порядку:

1. Подними базу данных в фоновом режиме:
   `docker compose up -d postgres`
2. Установи зависимости внутри контейнера приложения:
   `docker compose run --rm app pnpm install`
3. Сгенерируй Prisma Client:
   `docker compose run --rm app pnpm --filter web db:generate`
4. Примени миграции к базе данных:
   `docker compose run --rm app pnpm --filter web db:migrate`
5. Запусти всё приложение целиком:
   `docker compose up -d`
*(Теперь приложение доступно на порту 4000)*

## Шаг 6: Dev Tools (Скиллы)
В папке `skills/` находятся полноценные скиллы (инструкции в формате Markdown), скачанные с `skills.sh`. Ты (ИИ-агент) **обязан** использовать их для соблюдения архитектурных паттернов, TDD, дизайна и отладки.
1. Обязательно изучи содержимое папки `skills/` перед началом работы.
2. Используй их как источник истины при написании кода.
3. Если нужен скилл, которого нет в папке, используй скилл `find-skills` для поиска и установки через CLI `skills.sh`.
