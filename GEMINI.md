# HexaCash — Project Instructions for AI Assistant

## Memory Bank — обязательно читать в начале каждой сессии

В этом проекте используется **Memory Bank** — система постоянного контекста.  
**Перед тем как писать код или отвечать на вопросы по проекту**, прочитай следующие файлы в указанном порядке:

1. [`memory-bank/projectbrief.md`](./memory-bank/projectbrief.md) — полная спецификация продукта, функционал, ограничения, критерии успеха.
2. [`memory-bank/productContext.md`](./memory-bank/productContext.md) — зачем продукт, ЦА, happy path, ключевые разделы.
3. [`memory-bank/systemPatterns.md`](./memory-bank/systemPatterns.md) — архитектура, доменные сущности, воронка статусов, паттерны.
4. [`memory-bank/techContext.md`](./memory-bank/techContext.md) — технологический стек, интеграции, переменные окружения.
5. [`memory-bank/activeContext.md`](./memory-bank/activeContext.md) — текущий фокус, последние изменения, принятые решения.
6. [`memory-bank/progress.md`](./memory-bank/progress.md) — что сделано, что в очереди, известные проблемы.

## Правила работы с Memory Bank

- После значимых изменений в проекте — **обновляй соответствующие файлы Memory Bank**.
- Если что-то противоречит Memory Bank — уточни у пользователя, не игнорируй расхождение.
- `activeContext.md` и `progress.md` — обновляются чаще всего, отражают текущее состояние.
- `projectbrief.md` — источник истины по продукту; не противоречь ему без явного решения пользователя.

## Ключевые решения проекта (краткая выжимка)

- **Продукт:** B2B SaaS для компаний-продавцов на Avito (AI-ассистент + CRM + трекинг сделок).
- **Стек:** Next.js + Node.js (монорепо) + PostgreSQL + N8N + Docker.
- **AI-агент:** модель выбирается и конфигурируется в N8N (не хардкодится).
- **Avito:** все события (сообщения, заказы) приходят через N8N, не напрямую в приложение.
- **Статусы воронки:** расширяемые (не хардкодить), с event log и источником перехода.
- **Стадия:** планирование завершено → готово к инициализации монорепо.
