# Tech Context

## Технологический стек

| Слой | Технология |
|---|---|
| Фронтенд | Next.js, React, TypeScript, TailwindCSS |
| Бэкенд | Next.js API Routes (App Router) |
| ORM | Prisma |
| База данных | PostgreSQL |
| Автоматизации | N8N (self-hosted в Docker) |
| Уведомления | Telegram Bot API |
| Интеграция | Avito API (https://developers.avito.ru/api-catalog) |
| Инфраструктура | Docker + Docker Compose |
| Репозиторий | **Монорепо** (pnpm workspaces) |

## Требования для разработки

- Node.js (latest LTS)
- pnpm (пакетный менеджер, монорепо через pnpm workspaces)
- Docker + Docker Compose
- PostgreSQL (локально или в контейнере)
- N8N (self-hosted в Docker)

## Настройка локального окружения

1. Склонировать репозиторий.
2. Установить зависимости: `pnpm install`
3. Скопировать `.env.example` → `.env` и заполнить переменные.
4. Поднять инфраструктуру: `docker compose up -d`
5. Запустить режим разработки: `pnpm dev`

## Сторонние интеграции и API

- **Avito API** — https://developers.avito.ru/api-catalog  
  Авторизация: OAuth 2.0 (client_id / client_secret).  
  Используется для: объявления, сообщения, контакты, заказы, статусы доставки.  
  **Получение сообщений управляется N8N** — приложение получает события уже от N8N через webhook.

- **Telegram Bot API** — для отправки уведомлений по событиям (отправляется через N8N):
  - Нужен человек
  - Новая сделка
  - Новая сделка на исполнение
  - Сделка успешно закрыта

- **N8N** — self-hosted в Docker (в составе Docker Compose проекта).  
  Роль: **единый оркестратор** интеграций.  
  Через N8N: получение сообщений из Avito, вызов AI-агента, обновление карточек, Telegram, таймауты.

- **AI-агент** — модель **не фиксируется в коде**. Выбирается и конфигурируется в N8N.  
  Архитектурно поддерживает несколько агентов на компанию в будущем.

## Технические ограничения MVP

- Один AI-агент на компанию (UI). Архитектурно модель поддерживает несколько.
- Avito — единственный канал в MVP. Архитектурно поддерживаются дополнительные каналы.
- Переход в «Сделка оформлена» — только по данным из Avito, не по решению AI.
- Статусы должны быть реализованы расширяемо (не хардкодить нерасширяемую логику).
- Бэкенд реализуется через Next.js API Routes (App Router) — отдельного Node.js-сервиса нет.
- Prisma используется как единственный способ работы с БД (не raw SQL в приложении).
