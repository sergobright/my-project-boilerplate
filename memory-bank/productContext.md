# Product Context

## Зачем существует этот проект

**HexaCash** — B2B SaaS-платформа для компаний-продавцов на Avito. Автоматизирует обработку входящих сообщений, управление карточками продаж, работу с заказами и контроль исполнения сделок через Avito-доставку.

Продукт закрывает потребность в **едином рабочем месте**, где AI-агент ведёт переписку, а человек подключается только там, где это нужно.

## Проблемы, которые мы решаем

1. Слишком много ручной переписки с покупателями на Avito.
2. Потеря лидов из-за медленных ответов на сообщения.
3. Отсутствие единого интерфейса для контроля переписок, карточек и сделок.
4. Сложность сопровождения заказов с Avito-доставкой.
5. Непрозрачная логика: когда AI, а когда человек должен вести диалог.

## Целевая аудитория

**Малый и средний бизнес, который продаёт товары через Avito:**

- Предприниматели, которые сами ведут продажи.
- Небольшие команды продавцов (несколько человек).
- Компании с большим количеством объявлений и регулярным потоком диалогов.

**Кто вне фокуса MVP:**
- Enterprise с глубокой кастомизацией ролей.
- Омниканальные продажи (много каналов).
- Сложное постпродажное обслуживание.

## Как это должно работать

**Основной сценарий (happy path):**

1. Покупатель пишет в Avito → система создаёт карточку во «Входящих».
2. AI-агент читает переписку, отвечает, квалифицирует намерение.
3. AI переводит карточку в «Дожим» при явном интересе.
4. Если нужен человек — AI ставит флаг, карточка идёт в «Ожидание человека».
5. Пользователь перехватывает чат, выключает AI, ведёт диалог вручную.
6. Покупатель оформляет заказ через Avito-доставку → статус «Сделка оформлена» (только по данным из Avito).
7. Компания отправляет товар → «К отправке» → «В пути» → «Получено клиентом» → «Исполнено».
8. Telegram-уведомления по ключевым событиям на всём пути.

## Ключевые разделы продукта

| Раздел | Назначение |
|---|---|
| Kanban | Воронка продаж — все карточки по статусам |
| Чат | Удобная работа с переписками |
| Сделки | Карточки с `is_deal=true`, статусы исполнения |
| Объявления | Синхронизированные объявления из Avito |
| Контакты | Покупатели, история взаимодействий |
| Dashboard | Метрики по сделкам и деньгам |
| Настройки | Avito, Telegram, системный промпт, пользователи |

## Пользовательский опыт (UX)

- Простой интерфейс без перегруженных настроек.
- Пользователь видит только то, что нужно для работы прямо сейчас.
- Статусы меняются через разрешённые события (не произвольный drag-and-drop).
- AI и ручной режим чётко разграничены визуально.
- Telegram-уведомления, чтобы не сидеть в платформе постоянно.
- Два уровня доступа: корневая админка платформы и кабинет компании.
